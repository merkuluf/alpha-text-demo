# Markdown режим с форматом Jira

**Статус:** Реализовано | **Версия:** 1.0

## Обзор

Компонент `TiptapEditor` поддерживает два режима работы:

1. **Визуальный режим** - редактирование с живым превью форматирования
2. **Режим Текст (Markdown)** - редактирование в текстовом формате Jira

Оба режима полностью синхронизированы и используют формат Jira для markdown разметки.

## Форматирование текста

### Базовое форматирование

| Синтаксис | Результат | Кнопка |
|-----------|-----------|--------|
| `*жирный*` | **жирный** | B |
| `_курсив_` | *курсив* | I |
| `-зачеркнутый-` | ~~зачеркнутый~~ | S |
| `+подчеркнутый+` | <u>подчеркнутый</u> | U |
| `{{код}}` | `код` | Code |

### Применение форматирования

**В визуальном режиме:**
1. Выдели текст
2. Клик на кнопку форматирования (B, I, U, S, Code)
3. Текст обернётся в символы

**В режиме Текст (Markdown):**
1. Выдели текст в textarea
2. Клик на кнопку форматирования
3. Текст обернётся в символы Jira формата
4. Выделение включает сами символы форматирования

### Toggle поведение

Повторный клик на кнопку **отменяет** форматирование:

```
текст → *текст* → текст → *текст* ...
```

Это работает как в визуальном режиме, так и в режиме Текст.

## Заголовки

Заголовки используют синтаксис `hN. Текст` где N - уровень (1-5).

| Синтаксис | Результат |
|-----------|-----------|
| `h1. Заголовок` | # Заголовок |
| `h2. Заголовок` | ## Заголовок |
| `h3. Заголовок` | ### Заголовок |
| `h4. Заголовок` | #### Заголовок |

### Применение заголовков

**В визуальном режиме:**
- Меню "Стиль" → выбрать уровень заголовка
- Или выбрать текст и нажать на нужный заголовок

**В режиме Текст:**
- Выдели строку → выбрать нужный заголовок из меню "Стиль"
- Или вручную напиши `h1. `, `h2. ` и т.д.

### Toggle для заголовков

Если строка уже имеет заголовок, повторный клик его удаляет:

```
Текст → h1. Текст → Текст → h1. Текст ...
```

## Списки

### Маркированный список

```
* Элемент первого уровня
** Элемент второго уровня
```

**В режиме Текст:** нажми на кнопку "Списки" → "Маркированный список"

### Нумерованный список

```
# Элемент первого уровня
## Элемент второго уровня
```

**В режиме Текст:** нажми на кнопку "Списки" → "Нумерованный список"

### Toggle для списков

```
текст → * текст → текст → * текст ...
```

## Специальные блоки

### Блок кода

```
{code}
const hello = "world";
console.log(hello);
{code}
```

**В режиме Текст:** нажми на кнопку "{ }" (блок кода)

### Цитата (Blockquote)

```
bq. Это цитата или выделенный текст
```

**В режиме Текст:** нажми на кнопку "\"" (цитата)

### Горизонтальная линия

```
----
```

**В режиме Текст:** нажми на кнопку "─" (горизонтальная линия)

## Переключение режимов

### Кнопки переключения

В тулбаре справа есть две кнопки для переключения режимов:

- **Текст** - переходит в режим markdown (Jira формат)
- **Визуально** - переходит в визуальный режим редактора

### Синхронизация при переключении

При переключении между режимами происходит автоматическая конвертация:

**Визуальный → Текст:**
- `**жирный**` → `*жирный*`
- `*курсив*` → `_курсив_`
- Заголовки в стандартный Jira синтаксис

**Текст → Визуальный:**
- `*жирный*` → `**жирный**`
- `_курсив_` → `*курсив*`
- Заголовки парсируются в визуальные стили

## Техническая реализация

### Расширения Tiptap

Используются кастомные расширения для контроля синтаксиса markdown:

- **CustomBold** - использует `*текст*` вместо `**текст**`
- **CustomItalic** - использует `_текст_` вместо `*текст*`

Это сделано для избежания конфликтов парсера при одновременном применении нескольких форматирований.

### Функции форматирования

#### wrapMarkdownSelection(before, after)

Основная функция для применения форматирования к выделенному тексту.

**Особенности:**
- Проверяет есть ли уже форматирование вокруг текста
- Если есть - удаляет (toggle поведение)
- Если нет - добавляет символы
- Включает символы в выделение для удобства

#### applyHeadingLevel(level)

Применяет заголовок нужного уровня к текущей строке.

**Синтаксис:** `h1. `, `h2. `, `h3. `, `h4. `, `h5. `

#### markdownBulletList / markdownOrderedList

Применяют маркированный (`*`) или нумерованный (`#`) список.

## Примеры использования

### Сценарий 1: Форматирование в режиме Текст

```
1. Напиши текст: "это важное сообщение"
2. Выдели "важное"
3. Нажми B (Bold)
4. Результат: "это *важное* сообщение"
5. Нажми B еще раз
6. Результат: "это важное сообщение"
```

### Сценарий 2: Создание заголовка

```
1. Напиши: "Добро пожаловать"
2. Выдели строку
3. Нажми на "Заголовок 1"
4. Результат: "h1. Добро пожаловать"
5. Переключись на "Визуально"
6. Увидишь большой заголовок первого уровня
```

### Сценарий 3: Создание списка

```
1. В режиме Текст напиши:
   Первый элемент
   Второй элемент
   
2. Выдели первую строку
3. Нажми "Списки" → "Маркированный список"
4. Результат: "* Первый элемент"
5. Повтори для второй строки
6. Результат: "* Второй элемент"
```

## Известные особенности

### Toggle поведение

Выделение всегда включает символы форматирования:
- Выделил `*текст*` → при удалении удалятся символы, выделение останется на `текст`
- Это удобно для быстрого переключения между форматированиями

### Одновременное форматирование

Можно комбинировать форматирования:
```
*_жирный курсив_*
-+зачеркнутый подчеркнутый+-
```

### Вложенные заголовки

Заголовки работают на уровне строки:
```
h1. Заголовок первого уровня
h2. Заголовок второго уровня (не вложенный)
```

Это особенность Jira формата - все заголовки находятся на одном уровне вложенности.

## Интеграция в другие проекты

При импорте компонента как npm пакета, формат markdown можно настроить через пропсы (в будущих версиях).

Текущая версия поддерживает только Jira формат.

## Клавиатурные сокращения

В визуальном режиме работают стандартные сокращения:
- `Ctrl+B` - жирный
- `Ctrl+I` - курсив
- `Ctrl+U` - подчеркивание
- `Ctrl+E` - код
- `Ctrl+Z` - отменить
- `Ctrl+Y` - повторить

В режиме Текст сокращения **не работают** - используй кнопки тулбара.
